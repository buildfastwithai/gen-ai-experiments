# Contributing Content

## Goal

Keep repository content discoverable, consistent, and easy to run.

## Required For New Content

- Add a `README.md` in each new project folder.
- Include setup and run instructions.
- Include model/provider details.
- Add difficulty and status labels.

## Metadata Checklist

Each project README should include:

- What it is
- Who it is for
- Stack used
- Model/provider used
- Run steps
- Screenshots or demo link
- Tags
- Difficulty: beginner, intermediate, advanced
- Status: stable, experimental, archived

## Folder Placement Rules

- Use `apps/` for runnable projects.
- Use `cookbooks/` for model/provider notebooks and tutorials.
- Use `agents/` for agent-focused content.
- Use `rag/` for retrieval-focused content.
- Use `fine-tuning/` for fine-tuning guides.
- Use `workshops/` for guided sessions.
- Use `roadmaps/` for learning paths.
- Use `experiments/` for unstable or draft content.

## Promotion Rule

When experiments mature, move them to the best stable category.
