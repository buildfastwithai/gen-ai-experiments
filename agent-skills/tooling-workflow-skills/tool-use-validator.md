# Tool Use Validator

Validate function-call JSON payloads against schemas before execution.

## Use When
- Tool arguments are generated by an LLM.
- A user asks for schema-safe payload validation.
- Preventing runtime tool failures is critical.

## Inputs To Collect
- JSON schema (or equivalent constraints).
- Candidate payload.
- Unknown-field policy (allow/deny).
- Coercion policy (strict/permissive).

## Workflow
1. Parse and sanity-check schema.
2. Parse payload and perform type/constraint validation.
3. Validate required fields and nested paths.
4. Detect unknown fields and invalid enum values.
5. Return clear, path-aware validation output.

## Output Template
Use this exact template:

# Validation Result: VALID | INVALID

## Schema Summary
- Tool:
- Required fields:

## Payload Assessment
- [Field path] -> pass/fail

## Errors
- [path]: expected [X], got [Y], suggested fix [Z]

## Safe Payload
- Include only when VALID

## Quality Bar
- Never infer missing required values.
- Keep errors deterministic and machine-readable.


## Advanced Guidelines & Deep Dive
### Deep Validation Logic
- **Type Coercion Awareness**: Recognize that LLMs often send strings `"true"` instead of booleans `true`, or `"123"` instead of `123`. Provide rules on whether the validator should strictly reject or automatically coerce safely.
- **Enum Enforcement**: If a field expects `['production', 'staging']` and the LLM sends `'prod'`, reject it explicitly showcasing the allowed list.
- **Nested Object Traversal**: Validation must recursively check deep objects, not just top-level keys.

### Error Message Engineering
- **Contextual Syntax Help**: For invalid errors, provide the exact JSON path and a hint (e.g., `Error at payload.config.retry_count: expected integer, got array. Fix: provide a number between 1-5.`).
- **Missing Required Fields**: Do not attempt to hallucinatory-fill required arguments like `user_id`. Fail immediately.

### Anti-Patterns to Avoid
- **Leaking Internal State**: Don't echo back stack traces to the LLM agent; provide a JSON-friendly error envelope it can read programmatically.
- **Ignoring Additional Properties**: Accepting payloads with unexpected keys can lead to upstream poisoning. Strip or reject unknown keys gracefully.
