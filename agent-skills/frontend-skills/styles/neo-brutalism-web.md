# Neo-Brutalism Web Designer

Design and implement loud, high-contrast, hard-edged interfaces with intentional friction and personality.

## Use When
- A user asks for a neo-brutalist landing page or app UI.
- The goal is to stand out with raw, expressive visuals.
- The design should reject smooth, polished startup aesthetics.

## Inputs To Collect
- Brand attitude (playful, rebellious, artistic, activist).
- Content priority (hero, CTA, feature blocks, pricing).
- Acceptable intensity level (mild, medium, extreme).
- Technical stack constraints (React, Tailwind, vanilla CSS).

## Workflow
1. Define the visual manifesto: what rules this UI intentionally breaks.
2. Lock aggressive tokens: black/white base, 1-2 accent colors, thick border system.
3. Build asymmetrical layouts with oversized headings and uneven rhythm.
4. Add interaction states with snap motion and bold focus outlines.
5. Validate accessibility contrast and keyboard navigation.

## Quality Bar
- Hard edges preferred (0 radius unless justified).
- Borders must feel structural, not decorative.
- Type should carry hierarchy through size and weight.
- Keep semantic HTML and a11y intact.

## Advanced Guidelines & Deep Dive
### Visual Principles
- **Constructive Harshness**: Use thick outlines, offset shadows, abrupt color blocks, and unapologetic scale shifts.
- **Typography as Structure**: Headings are architectural elements, not just labels. Use dramatic contrast in size and weight.
- **Asymmetry with Intent**: Offset cards, broken grid columns, and uneven gutters are acceptable when hierarchy remains clear.

### Motion Language
- **Snappy Transitions**: Favor short, crisp transitions over soft easing. UI should feel mechanical and direct.
- **Tactile Hover States**: Add small positional shifts, hard shadow swaps, or border inversion to signal interaction.

### Anti-Patterns to Avoid
- **Softening Everything**: Rounded corners and blurred shadows can dilute brutalist character.
- **Random Chaos**: Noise without hierarchy hurts usability. Every aggressive choice must still guide scanning.
- **Illegible Extremes**: Oversized display type still needs readable line length and contrast compliance.
